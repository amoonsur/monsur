1. Using JIRA (HelpDesk ticket system)

Skip to end of metadata
  • Created by Brian Wade, last modified on Mar 16, 2016
Go to start of metadata

We utilize JIRA for our development tickets and to communicate issues/problems and discussions with you. Once your account is created, you will begin to receive email notifications about new tickets that are created and open for development. You can click on on the link from the email to go directly to the task to review if you want to work on it, or you can login to JIRA and look for open tickets.

 

  • In order to see the open tickets, go to Issues > Open Tickets. Here we can see the tickets which are available to be claimed/worked. It is very important to note the direction which the tickets should be worked. The tickets should be worked from the top (Highest Priority) to the bottom (Lowest Priority). It's ok to skip a ticket or two if your skills are not appropriate skill level, but please still work from top down.


  • Once you've reviewed some tickets and found the one that you'd like to work, please assign the ticket to yourself:
  • So now that you have the ticket assigned and you are ready to perform the work on it, you will need to make sure that you have access to the GIT repository. As a new user, you will not have access to any repositories. You will need to request access to the particular repository from Brian Wade via Skype. The name of the repository will always be part of the ticket, so for example if ticket is HBAKE-34 then you will need to request access to the HBAKE git repository.

    Please continue to 2. Cloning and Setting up your local GIT repository
